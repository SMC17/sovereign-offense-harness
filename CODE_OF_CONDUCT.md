# Code of Conduct

This is a small, single-author OSS project. The rules are short on
purpose.

## Standard

- **Be civil.** Disagree with the argument, never the person.
- **Keep it technical.** Code, math, design tradeoffs, evidence.
  Off-topic political / identity arguments belong elsewhere.
- **No harassment.** In issues, PRs, the commit history, or
  out-of-band channels that reference this project. Includes
  derogatory comments about people, demographics, beliefs, or
  affiliations.
- **No spam or recruitment churn.** Issues are for technical work.

We follow the *spirit* of the [Contributor Covenant
v2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/)
without adopting it formally — the goal is a calm, evidence-driven
collaboration surface.

## Scope

This applies anywhere the project is the context:

- Issues, pull requests, commits, code review.
- Documentation, examples, error messages.
- Project social media accounts when they exist.
- Adjacent forum / chat threads where someone is representing this
  project.

## Enforcement

Single-author enforcement model — there is no separate moderation
team. Reports go to the contact listed in
[`SECURITY.md`](SECURITY.md).

Possible responses, in escalation order:

1. **Private note** asking for a change in conduct.
2. **Public correction** on the thread.
3. **Lock** the issue or PR.
4. **Block** the contributor from the repository.
5. **Permanent block** on repeated or severe violations.

The author reserves the right to skip steps for clear-cut cases
(threats, doxxing, credential theft attempts, etc.).

## Maintainer's promise

The same standard applies to me. If you think I've violated it,
email the SECURITY.md contact. Misconduct from a maintainer is worth
more attention than the same conduct from a contributor, not less.
